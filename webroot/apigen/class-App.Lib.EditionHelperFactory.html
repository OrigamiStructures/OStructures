Class EditionHelperFactory

FactoryHelper swaps Edition and Format helper subclasses onto a common call point

Edition and Format display and form output follow a variety of rules depending on the Edition type. All the view and fieldset elements are standardized and they all call a single helper variable for service. So the underlying helper class must be managed so the correct rule set is used.

protected array $_concrete_helper

Keyed access to concrete Edition helpers

Keyed access to concrete Edition helpers

Different edition types need different output services for their parts. Concrete flavors of helpers are stored for each type.

In this uninitialized state each key contains a string. This indicates the need to load the concrete helper. The string is the key to _concrete_build_map which tells which edition types share flavors of concrete helper.

Once initialization is done for a specific type (and its partners) it will hold a helper object rather than a string.

EditionFactory->concrete() is the accessor method to fetch these helpers. That class will lazy-load the objects too.
